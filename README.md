

# Q-DMDA

Clean release code for Q-DMDA, a quaternion geometric domain adaptation method for cross-subject motor imagery EEG classification.

This package contains runnable training code for the Blankertz and 2003004 datasets, together with the final precomputed compact quaternion-GFK feature files.

## Environment

Recommended environment:

```text
OS: Windows 10/11 or Linux
Python: 3.10
GPU: NVIDIA GPU with a CUDA-compatible PyTorch build
```

The current training scripts use CUDA in the model and tensor calls, so a CUDA-enabled PyTorch environment is recommended for direct execution. CPU-only execution requires manually replacing the CUDA calls with device-agnostic code.

Create an environment with Conda:

```bash
conda create -n qdmda python=3.10 -y
conda activate qdmda
```

Install PyTorch according to your local CUDA version, then install the remaining dependencies:

```bash
pip install -r requirements.txt
```

Main Python packages:

```text
torch
numpy
scipy
h5py
scikit-learn
einops
```

MATLAB is not required for running this release because the final feature files are already included.

## Repository Layout

```text
Q-DMDA/
  main_MI1.py                  # Blankertz training entry
  main_MI3.py                  # 2003004 training entry
  data.py                      # LOSO feature loading and dataloaders
  losses.py                    # label smoothing, marginal MMD, conditional MMD
  adversarial.py               # gradient reversal layer
  core_qnn/                    # quaternion neural network layers
  features/
    Blankertz/                 # final Blankertz compact features
    2003004/                   # final 2003004 compact features
  requirements.txt
```

Generated checkpoints, logs, prediction CSV files, and Python caches are not included in the release.

## Included Feature Files

The included feature files are LOSO compact quaternion-GFK features.

Blankertz:

```text
features/Blankertz
7 LOSO files
raw k-axis tangent-space dimension: 7260
used dimension: 512
```

2003004:

```text
features/2003004
5 LOSO files
raw k-axis tangent-space dimension: 28920
used dimension: 512
```

Each feature file follows this naming pattern:

```text
DATASET_LOSO_target_XX_spat_W4_logmap_gfk_k25.mat
```

The feature files are loaded directly by `data.py`. No raw EEG files or feature-engineering scripts are needed for the released experiments.

## Quick Start

From the repository root:

```bash
cd Q-DMDA
```

## Run Blankertz

Single seed:

```bash
python main_MI1.py
```

Five seeds:

```bash
python main_MI1.py --seeds 1,2,3,4,5
```

The Blankertz entry is configured with:

```text
targets = 7,6,5,4,3,2,1
num_source_domains = 6
model_feat_dim = 512
```

## Run 2003004

Single seed:

```bash
python main_MI3.py
```

Five seeds:

```bash
python main_MI3.py --seeds 1,2,3,4,5
```

The 2003004 entry is configured with:

```text
targets = 5,4,3,2,1
num_source_domains = 4
model_feat_dim = 512
n_groups = 32
raw_attn_heads = 8
```

## Output Files

Training creates local result files such as:

```text
snapshot_seed*.txt
five_seed_summary.csv
checkpoints/*.pth
```

These files are ignored by `.gitignore` and can be regenerated by rerunning the scripts.

## Notes

- Target true labels are used only for evaluation.
- Fine-tuning uses source supervised loss and high-confidence target pseudo labels.
- Feature files are precomputed using unsupervised compact PCA before GFK alignment.
- No class labels or target true labels are used during compact projection or GFK alignment.
- The default single-seed run uses seed 1; use `--seeds 1,2,3,4,5` for the five-seed protocol.
